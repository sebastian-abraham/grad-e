# Teacher Flow Implementation

This document details the architecture, file structure, and deep implementation logic of the Teacher Flow feature, enabling educators to create and manage exams, utilize AI for grading criteria, perform seating arrangements, and manually review graded results.

## Data Models

### 1. `Exam` (`backend/models/Exam.js`)
Stores metadata and AI-generated criteria for an assessment.
- **Header Fields**: `name` (String), `subjectId` (ObjectId), `classId` (ObjectId), `teacherId` (ObjectId), `date` (Date), `totalMarks` (Number).
- **`status`**: Enforced enum `["Draft", "Setup Complete", "Sheets Uploaded", "Processing", "Graded"]`.
- **`criteria`** (Array): Stores objects generated by Gemini: `{ questionNumber, prompt, marks, valuationNotes }`.
- **`seatingArrangement`**: Object containing matrix dimensions `rows` and `cols` (Numbers), and `assignments` (Array mapping `row`, `col` to `studentId`).

### 2. `Submission` (`backend/models/Submission.js`)
Represents an individual student's test uploaded against an Exam.
- **Relational Links**: `examId` (ObjectId) and `studentId` (ObjectId - optionally null until explicitly assigned).
- **File Data**: `fileName` (String) and `pdfData` (String - Base64 encoding).
- **`status`**: Enum `["Ungraded", "Graded"]`.
- **`feedback`** (Array): Maps directly to Exam criteria, adding `pointsAwarded`, `studentAnswer`, `teacherFeedback`, `status` (`correct/partial/incorrect`), and `flags`.

## Backend API Endpoints (`backend/routes/examRoutes.js`)

### Criteria Generation via AI
**`POST /api/exams/:id/generate-criteria`**
- Intercepts uploads via `multer` configured for memory storage and PDF validation.
- Extracts `questionPaper` and `answerKey` buffers natively as Base64 strings.
- Calls `https://generativelanguage.googleapis.com/v1beta/models/gemini-2.5-flash:generateContent`.
- **Prompt Logic**: Commands Gemini to explicitly act as a curriculum designer parsing the two PDFs and formulating a strict JSON array representing Question Numbers, Prompts, Marks, and Valuation notes.
- Extracts the stringified JSON and automatically executes `Exam.findByIdAndUpdate` saving it directly.

### Mock Auto-Grading Pipeline
**`POST /api/exams/:id/grade-all`**
- Instantly sets Exam status to "Processing".
- Wraps logic in a `setTimeout(..., 3000)` representing a mock background job.
- Iterates existing "Ungraded" Submissions, generates a randomized score (20-100), and populates `feedback` array with static placeholder text "Mocked Answer Extracted". 
- Upgrades Exam to "Graded".

### Exam Submissions CRUD
- `POST /:id/submissions`: Takes bulk array of `sheets`, casting to Base64 strings and running `Submission.insertMany()`.
- `GET /:id/submissions`: Intentionally strips `pdfData` by invoking `toObject()` and explicitly dropping the heavy payload via `delete obj.pdfData` to prevent crashing the frontend browser on list renders.

## Frontend UI Components

### 1. The Creation Wizard (`src/pages/CreateExam.jsx`)
- Uses React state (`step` 1 or 2) to manage standard input metadata vs AI generation.
- **Step 1**: Forms require `subjectId` and `classId` fetched actively from `teacherRoutes.js`.
- **Step 2**: Pushes JavaScript `File` native objects mapped via `FormData` natively over fetch to the `/generate-criteria` endpoint. Maps the returned AI JSON strictly to state enabling inline text editing before confirming to DB.

### 2. The Command Hub (`src/pages/ExamDetail.jsx`)
Managed heavily via `activeTab` states. Include three distinct architectural spaces:
- **`AnswerSheetsTab`**: Contains the bulk `multiple` input upload handler. Includes a fallback student `<select>` assignment mapping array matching the linked Class Roster for unmatched papers.
- **`SeatingTab`**: Implements a `gridTemplateColumns: repeat(${cols}, 1fr)` dynamic CSS styling grid. Clicks check existing array `assignments` mapping visually translating `onMouseEnter` scale micro-animations. Prevents assigning if all students exhausted.
- **`OverviewTab`**: Aggregates `Chart.js` via `react-chartjs-2`, counting custom score bucket ranges ("0-20", "21-40", etc.) and rendering a colored `<Bar>` visualization alongside static average/highest metrics.

### 3. Split Grading Review (`src/pages/GradingView.jsx`)
- **Left Panel**: Injects the base64 `submission.pdfData` directly over an `<iframe src="data:application/pdf;base64,...">` scaling natively to browser rendering engines.
- **Right Panel**: A scrollable vertical Form looping over `feedback`. Modifying the manual override `pointsAwarded` input triggers `handleScoreChange` cascading automatically shifting the global `.reduce()` aggregation of the final score while contextually painting background cards `ecfdf5` (green/correct) vs `fef2f2` (red/incorrect) instantly communicating logic.
